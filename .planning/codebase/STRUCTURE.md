# Codebase Structure

**Analysis Date:** 2026-02-08

## Directory Layout

```
[project-root]/
├── main/                    # Primary application source
│   ├── app_main.c           # Boot orchestration and entry point

│   ├── thermostat/          # UI subsystem
│   ├── connectivity/        # Network and MQTT
│   ├── sensors/             # Environmental and presence
│   ├── streaming/           # WebRTC camera/mic
│   ├── assets/              # Fonts, images, audio blobs

│   ├── esp_http_server/     # HTTP server implementation
│   ├── av_render/           # Audio/video rendering
│   └── esp32_p4_nano/       # Waveshare BSP (legacy)
├── managed_components/      # ESP-IDF component manager cache
├── scripts/                 # Build and utility scripts
├── assets/                  # Source assets (fonts, audio, images TOML)
├── docs/                    # Documentation

├── partition_table/         # Flash partition definitions
├── CMakeLists.txt           # Project root build file
├── sdkconfig.defaults       # Default ESP-IDF configuration
├── sdkconfig.defaults.local # Local overrides (gitignored)

```

## Directory Purposes

**main/:**
- Purpose: Core application implementation
- Contains: All C source and headers for thermostat logic
- Key files: `app_main.c`, `thermostat_ui.c`, `CMakeLists.txt`

**main/thermostat/:**
- Purpose: UI rendering and interaction
- Contains: LVGL widgets, theme, animations, input handling
- Key files: `ui_state.h`, `ui_theme.c`, `ui_setpoint_input.c`, `backlight_manager.c`

**main/connectivity/:**
- Purpose: Network stack and communication protocols
- Contains: MQTT client, Wi-Fi proxy, HTTP server, OTA, time sync
- Key files: `mqtt_manager.c`, `mqtt_dataplane.c`, `wifi_remote_manager.c`, `http_server.c`

**main/sensors/:**
- Purpose: Hardware sensor interfaces
- Contains: AHT20/BMP280 wrapper, LD2410 radar presence
- Key files: `env_sensors.c`, `radar_presence.c`

**main/streaming/:**
- Purpose: WebRTC media streaming (optional, gated by `CONFIG_THEO_CAMERA_ENABLE`)
- Contains: WebRTC orchestration, WHEP signaling, microphone capture
- Key files: `webrtc_stream.c`, `whep_endpoint.c`

**main/assets/:**
- Purpose: Compiled binary assets
- Contains: LVGL font files, image data, PCM audio
- Generated by: `scripts/generate_fonts.py`, `generate_images.py`, `generate_sounds.py`
- Key files: `fonts/figtree_*.c`, `images/*.c`, `audio/*.c`

**components/:**
- Purpose: Custom ESP-IDF components not from registry
- Contains: Modified esp_http_server, av_render component, legacy BSP
- Key files: `esp_http_server/src/httpd_*.c`, `av_render/src/av_render.c`

**scripts/:**
- Purpose: Build automation and utilities
- Contains: Asset generators, OTA push, worktree management
- Key files: `generate_fonts.py`, `generate_images.py`, `generate_sounds.py`, `push-ota.sh`

**assets/ (root):**
- Purpose: Source definitions for binary assets
- Contains: TOML config files for fonts, images, audio
- Key files: `fonts/fontgen.toml`, `images/imagegen.toml`, `audio/soundgen.toml`

## Key File Locations

**Entry Points:**
- `main/app_main.c`: Application boot sequence (609 lines)

**Configuration:**
- `CMakeLists.txt`: Project build configuration
- `sdkconfig.defaults`: Default ESP-IDF Kconfig values
- `main/idf_component.yml`: Component manifest with LVGL 9.4, MQTT, ESP-Hosted
- `main/CMakeLists.txt`: Source file lists and conditional compilation

**Core Logic:**
- `main/thermostat/ui_state.h`: Global view model definition
- `main/connectivity/mqtt_dataplane.c`: MQTT message processing
- `main/thermostat/ui_setpoint_input.c`: Touch gesture handling
- `main/sensors/env_sensors.c`: Sensor polling and telemetry

**UI Components:**
- `main/thermostat/ui_top_bar.c`: Weather, room, HVAC widgets
- `main/thermostat/ui_setpoint_view.c`: Temperature slider rendering
- `main/thermostat/ui_actions.c`: Mode/power/fan controls
- `main/thermostat/ui_splash.c`: Boot progress display

**Headers (Public Interfaces):**
- `main/thermostat_ui.h`: UI attach/refresh API
- `main/thermostat/ui_state.h`: View model and constants
- `main/connectivity/mqtt_dataplane.h`: Command publishing API
- `main/sensors/env_sensors.h`: Sensor readings API

## Naming Conventions

**Files:**
- C source: `snake_case.c` (e.g., `mqtt_dataplane.c`)
- Headers: `snake_case.h` (e.g., `ui_state.h`)
- Asset files: `descriptive_name.c` (e.g., `figtree_tnum_semibold_120.c`)

**Functions:**
- UI module prefix: `thermostat_*` (e.g., `thermostat_ui_attach()`)
- Connectivity prefix: `*_manager_*` or module-specific (e.g., `mqtt_manager_start()`)
- Sensor prefix: `env_sensors_*`, `radar_presence_*`
- Static helpers: `snake_case()` without prefix

**Types:**
- Structs: `snake_case_t` (e.g., `thermostat_view_model_t`)
- Enums: `snake_case_t` with prefixed values (e.g., `thermostat_target_t`)
- Config structs: `*_config_t` suffix

**Globals:**
- View model: `g_view_model`
- Font bundle: `g_fonts`
- LVGL objects: `g_root_screen`, `g_layer_top`
- Flags: `g_ui_initialized`

**Kconfig Options:**
- Prefix: `CONFIG_THEO_*`
- Examples: `CONFIG_THEO_CAMERA_ENABLE`, `CONFIG_THEO_RADAR_ENABLE`

## Where to Add New Code

**New UI Widget:**
- Implementation: `main/thermostat/ui_<feature>.c`
- Header: `main/thermostat/ui_<feature>.h`
- Registration: Call from `thermostat_ui_init()` in `main/thermostat_ui.c`

**New Connectivity Feature:**
- Implementation: `main/connectivity/<feature>.c`
- Header: `main/connectivity/<feature>.h`
- Init: Call from `app_main.c` boot sequence

**New Sensor:**
- Implementation: `main/sensors/<sensor>.c`
- Header: `main/sensors/<sensor>.h`
- Integration: Call init from `app_main.c`, publish via existing MQTT

**New Assets:**
- Source: Add to appropriate TOML in `assets/`
- Regenerate: Run `scripts/generate_*.py`
- Include: Add LV_IMG_DECLARE or extern to using file

**Utilities:**
- Shared helpers: `main/thermostat/ui_helpers.c` (existing)
- New module: Create in `main/` subdirectory matching concern

## Special Directories

**managed_components/:**
- Purpose: ESP-IDF Component Manager download cache
- Generated: Yes (by `idf.py build`)
- Committed: No (in `.gitignore`)
- Contains: LVGL, ESP-Hosted, MQTT, etc. at pinned versions

**build/:**
- Purpose: CMake build artifacts
- Generated: Yes (by `idf.py build`)
- Committed: No (in `.gitignore`)
- Contains: Object files, firmware binaries, flashable images

**main/build/:**
- Purpose: idf.py log output
- Generated: Yes
- Committed: No

**tmp/:**
- Purpose: Temporary files, worktrees
- Generated: Yes
- Committed: No

**scratch/:**
- Purpose: Experimental Arduino sketches and prototypes
- Generated: No
- Committed: Yes
- Examples: `falling_cool/`, `hearth_glow/`, `rainbow_loop/`

**worktrees/:**
- Purpose: Git worktrees for parallel development
- Generated: By `scripts/init-worktree.sh`
- Committed: No
